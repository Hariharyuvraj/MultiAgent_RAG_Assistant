  6. Data Models — schemas/state.py

  Three Pydantic models that flow through the entire pipeline:

  QueryInput
    ├── query: str                     ← raw user question
    ├── session_id: str                ← identifies conversation session
    └── timestamp: str

  ChunkResult
    ├── content: str                   ← text of retrieved chunk
    ├── source: str                    ← PDF filename
    ├── page: int                      ← page number
    └── score: float                   ← similarity score

  AgentState
    ├── query: str                     ← current user query
    ├── session_id: str
    ├── history: List[dict]            ← prior conversation turns
    ├── enriched_query: str            ← query + context from ContextAgent
    ├── plan: List[str]                ← sub-steps from PlannerAgent
    ├── retrieved_chunks: List[ChunkResult]  ← from SearchAgent
    ├── answer: str                    ← from AnalystAgent
    ├── sources: List[str]             ← cited sources
    ├── grounding_score: float         ← from GuardAgent
    ├── retry_count: int               ← how many retries happened
    └── passed_guard: bool             ← final gate

  7. Provider Factories

  providers/llm_factory.py

  Reads settings.yaml → returns correct LLM client:

  if provider == "groq"      → return ChatGroq(model=model_name, api_key=GROQ_API_KEY)
  if provider == "gemini"    → return ChatGoogleGenerativeAI(model=model_name, api_key=GOOGLE_API_KEY)
  if provider == "huggingface" → return HuggingFaceEndpoint(...)

  providers/embedding_factory.py

  Reads settings.yaml → returns correct embeddings:

  if provider == "huggingface" → return HuggingFaceEmbeddings(model_name=model_name)
  if provider == "google"      → return GoogleGenerativeAIEmbeddings(model=model_name)
  if provider == "openai"      → return OpenAIEmbeddings(model=model_name)

  One config line change switches the entire system. No code touched.

  8. Data Pipeline — Full Flow

  Step 1: Drop PDFs into documents/

  documents/
  ├── paper1.pdf
  ├── paper2.pdf
  └── notes.txt

  Step 2: ingest/doc_loader.py

  For each file in documents/:
    if .pdf  → LangChain PyPDFLoader → extract pages as Documents
    if .txt  → LangChain TextLoader → extract as Document

  Each Document has:
    page_content: str     ← raw text
    metadata:
      source: filename
      page: page_number
      format: pdf/txt

  Step 3: ingest/text_splitter.py

  LangChain RecursiveCharacterTextSplitter
    chunk_size    = 800    (from config)
    chunk_overlap = 100    (from config)

  Input:  List[Document] (full pages)
  Output: List[Document] (smaller chunks with metadata preserved)

  Example: 10-page PDF → ~40-60 chunks depending on density

  Step 4: ingest/embedder.py

  1. Load embedding model via embedding_factory.py
  2. For each chunk:
     - Call embed_documents([chunk.page_content])
     - Returns 384-dim vector (HF) or 768-dim vector (Google)
  3. Store in ChromaDB:
     - collection: "insight_docs"
     - persist_path: ./storage/vectordb
     - Stored: vector + raw text + metadata
  4. Print summary: "Indexed N chunks from M documents"

  Full Ingest CLI Command:

  python cli.py --ingest

  9. The 5 Agents — Detailed

  Agent 1: ContextAgent (agents/context_agent.py)

  Role: Manages conversation memory. Determines if the new query relates to prior turns.

  Input: AgentState with query and history

  Logic:
  1. Load last N turns from state.history (N from config: max_history_turns=10)
  2. Format history as: "User: ... Assistant: ..."
  3. If history is not empty:
     - Ask LLM: "Is this query a follow-up to the conversation? YES/NO"
     - If YES → prepend relevant history context to query
     - If NO  → treat as fresh query
  4. Set state.enriched_query = (original query + relevant context)

  Output: AgentState with enriched_query filled

  ADK Role: First node in SequentialAgent pipeline

  ---
  Agent 2: PlannerAgent (agents/planner_agent.py)

  Role: Breaks the enriched query into a retrieval execution plan.

  Input: AgentState with enriched_query

  Logic:
  System Prompt:
    "You are a query planning expert. Given a user question about documents,
     break it into 2-4 specific retrieval sub-steps. Each step should be a
     focused search query. Return as numbered list."

  User Prompt: state.enriched_query

  LLM returns:
    1. Search for definition of [topic A]
    2. Find details about [topic B]
    3. Look for comparisons between [A] and [B]

  Parse numbered list → state.plan = [step1, step2, step3]

  Output: AgentState with plan filled (list of sub-queries)

  ADK Role: Second node in pipeline

  ---
  Agent 3: SearchAgent (agents/search_agent.py)

  Role: Executes each plan step as a vector search against ChromaDB.

  Input: AgentState with plan

  Logic:
  1. Load ChromaDB collection via LangChain retriever
  2. For each sub-query in state.plan:
     a. Embed the sub-query using embedding_factory
     b. Run similarity search: top_k=5, score_threshold=0.35 (from config)
     c. Collect ChunkResult(content, source, page, score)
  3. Deduplicate chunks (same source + page → keep highest score)
  4. Sort all chunks by score descending
  5. state.retrieved_chunks = top N unique chunks

  Output: AgentState with retrieved_chunks filled

  ADK Role: Third node in pipeline

  ---
  Agent 4: AnalystAgent (agents/analyst_agent.py)

  Role: Reads the retrieved chunks and builds the final answer with citations.

  Input: AgentState with enriched_query and retrieved_chunks

  Logic:
  System Prompt:
    "You are a document analyst. Answer the user's question using ONLY
     the provided document excerpts. Always cite the source document
     and page number. If the answer is not in the documents, say so."

  User Prompt:
    Question: {state.enriched_query}

    Documents:
    [1] Source: paper1.pdf, Page 3
        Content: {chunk1.content}

    [2] Source: paper2.pdf, Page 7
        Content: {chunk2.content}

    ...

  LLM returns structured answer with inline citations

  Parse response:
    state.answer  = full answer text
    state.sources = list of cited source filenames + pages

  Output: AgentState with answer and sources filled

  ADK Role: Fourth node in pipeline

  ---
  Agent 5: GuardAgent (agents/guard_agent.py)

  Role: Validates that the answer is actually grounded in the retrieved chunks. Prevents hallucination.

  Input: AgentState with answer and retrieved_chunks

  Logic:
  guardrails/grounding_check.py scores the answer:

    Method:
    1. Embed the answer
    2. Embed each retrieved chunk
    3. Compute cosine similarity between answer and each chunk
    4. grounding_score = max similarity across all chunks

    If grounding_score >= 0.60 (config threshold):
      state.passed_guard = True
      return answer to user

    If grounding_score < 0.60:
      state.retry_count += 1
      If retry_count < max_retries (2):
        Loop back to SearchAgent with broader query
      Else:
        Return fallback_message from config

  Output: Final answer delivered to user OR retry triggered

  ADK Role: Fifth node — has conditional branch (pass/retry/fallback)

  ---
  10. ADK Pipeline — graph/agent_graph.py

  Google ADK SequentialAgent:

  Pipeline:
    Runner
      └── SequentialAgent("insight_pipeline")
            ├── Node 1: ContextAgent
            ├── Node 2: PlannerAgent
            ├── Node 3: SearchAgent
            ├── Node 4: AnalystAgent
            └── Node 5: GuardAgent
                  ├── [PASS]  → return AgentState.answer
                  ├── [RETRY] → back to Node 3 (SearchAgent)
                  └── [FAIL]  → return fallback_message

  ADK Session:
    - session_id passed per conversation
    - InvocationContext carries AgentState between nodes
    - ADK handles state persistence within session

  ---
  11. Guardrails — guardrails/grounding_check.py

  Function: compute_grounding_score(answer, chunks)

  Steps:
  1. Use same embedding model (from embedding_factory)
  2. Embed answer string → answer_vector
  3. For each chunk:
     - Embed chunk.content → chunk_vector
     - Compute cosine_similarity(answer_vector, chunk_vector)
  4. Return max similarity as grounding_score (0.0 to 1.0)

  Threshold logic (in GuardAgent):
    >= 0.60 → PASS
    < 0.60 and retries < 2 → RETRY
    < 0.60 and retries >= 2 → FALLBACK

          │       → sets state.retrieved_chunks

                  → if score >= 0.60:

                  │     return answer + sources to user
                  │     trace_logger writes eval/traces/xxx.json
                  │
                  └── if score < 0.60 and retries < 2:
                        state.retry_count += 1
                        loop back to SearchAgent

                      if retries >= 2:
                        return fallback_message
